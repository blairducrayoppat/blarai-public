**Five "lessons about the lessons" a governance assessor would find remarkable:**

1. **Writing a lesson down installs nothing (L196).** "A class named in May recurred six more times in June." The corpus proved its own core thesis — that documentation is not a control — and responded structurally (the third-instance rule), which is the AIGP-grade move.
2. **The discipline records its own failures inline.** The recurrence-tally + `THIRD INSTANCE` convention makes the corpus *auditable against itself*: you can count how many times each class bit and whether it was ever structurally closed. A "clean" lessons list would hide exactly this. 4. **The instruments became the subject.** Roughly the last quarter of the corpus is about grading the graders — positive/negative controls, censored-data statistics, sensor placement out of the model's reach. A maturity signal: the failure surface moved up a level from product to meta-tooling.

## 5. TIER PROPOSAL — make the pre-mint check cheap, keep numbers permanent, archive not delete

**The problem to solve:** Rule 2 forces a full-file search before minting; the file is now ~81K tokens (~40% of a session's context). The search surface must shrink to the class *titles*, not the full text.

**Proposed three-tier layout (honors: numbers permanent · nothing deleted · archive-only growth):**

- **HOT — `LESSONS.md` (target ≈ 12–15K tokens).** Four sections only: (a) the **Rules**; (b) the existing **Canonical Tier C1–C20**, refreshed to fold in lessons 196–284 (currently unclassified); (c) a new **"Canon-32"** block carrying the *full text* of the §2 lessons (~32 × ~1,200 chars ≈ 9K tokens); (d) the **one-line index of all 284** (number + bolded title + tags + `↺k`/`✓control` flags). The one-line index (≈ 284 × ~90 chars ≈ 6–7K tokens) IS the pre-mint search surface and lives in the hot file.
- **INDEX — could be a section of the hot file (above) or a split `LESSONS_INDEX.md`.** One line per lesson: `N. **Title** — [tags] · ↺k · control:yes/no → archive#N`. This is the searchable SSOT the third-instance rule reads (recurrence count is a field, not a grep of prose).
- **ARCHIVE — `docs/lessons/LESSONS_ARCHIVE.md` (or per-lesson files).** The full text of all 284 lessons *with every recurrence tally and `(control:…)` marker*, append-only, never trimmed — the ~78K-token corpus, loaded only when a session needs one specific lesson's full story (grep to the number, read that block). Numbers permanent by construction.
- **`FIELD_NOTES.md`** absorbs the pure-mechanic content (80, 127, 161) and the mechanic-halves of the ~19 `†` lessons, each linked from its numbered lesson (Rule 4).

**The new mint workflow (and its cost):** search the **index** (title-only, already in the hot file → **~0 marginal tokens**, worst case grep of a ~7K-token `LESSONS_INDEX.md`) → match a class? append a dated tally to that number in the **archive** (open only that file) → no match? assign the next number in index + archive. **Pre-mint check cost: ~81K → ≤ ~7K tokens (≈ 91% reduction), and ~0 if the hot file is already resident.** The Canon-32 gives a reviewer the portfolio "page to read first" without opening the archive at all.
