## 1. ERA

**Dates covered: 2026-06-10 → 2026-06-16** (32 entries; one same-day cluster of 4 on 06-10, four on 06-12, seven across 06-13/14).

This is **THE air-gap-removal era — the single most consequential governance arc in the project's life.** In this span BlarAI crosses the line it was built never to cross: on 2026-06-12, with the Lead Architect at the screen, a system that had been *totally air-gapped for its entire existence* reaches the public internet for the first time, on purpose, through one Policy-Agent-gated door, and comes back clean ("FETCHED ok: http=200, bytes=89219"). Everything in the chunk orbits that milestone. The front half lands the UC-002/UC-003 program (~11,300 lines: the encrypted knowledge bank, the trafilatura article cleaner, the ingest UX, and the governance ADRs that bind them) and builds the containment boundary the egress *requires* — hostile fetched HTML is parsed inside a NIC-less Alpine Hyper-V guest across an AF_HYPERV vsock channel, never in the host process holding the unsealed encryption key. The back half is a rapid-fire sequence of go-live ceremonies (URL ingest, markdown extraction) and the immediate next builds — UC-003 display-only images and UC-010 image generation — both shipped fully *dormant behind weld locks*, establishing the era's signature discipline: **build the hardened code before the door opens, because "dormant" is never "safe to defer."** It is a security-by-design masterclass told through a chain of near-misses, honest failures, and one genuinely irreversible milestone.

## 2. GEMS

1. **2026-06-12 — The first packet off the box** — The air-gap milestone. Defensibility captured in one line: *"The green signal and the welded-shut state are the same fact viewed from two ends."* And the honesty that makes it portfolio-grade: he did NOT globally arm egress_guard for the first shot and said so — *"I would rather ship a true green with two named gaps than a green that papered over them."* One door (only module importing httpx), PA-adjudicated single-host allowlist, self-rewelding in a `finally`. `[governance/security][engineering][human-AI-collaboration]`

2. **2026-06-14 — The knowledge bank that could only be written to** — Two individually-correct security controls collided into a category error: provenance-is-not-trust (grounding recalled articles as UNTRUSTED_EXTERNAL) + the Stage-5 cosine leakage detector = *every faithful recall of a saved article looked like an exfiltration leak.* "You could save anything. You could get nothing back." Rejected the fast fix (trust-promotion) because it silently drops the action-lock; built a fourth provenance tier UNTRUSTED_KNOWLEDGE as the narrowest possible cut — one line changed, exempt from exactly one feed. `[failure-story][governance/security][engineering]`

3. **2026-06-12 — The door that would have tripped its own alarm** — The armed egress kill-switch and the door's own pre-fetch SSRF DNS resolution are each correct and *together fatal*: the lookup of the not-yet-allowlisted host is itself off-allowlist egress, so the operator's very first `/ingest` would have denied the fetch AND air-gapped the box into a re-arm ceremony. Rejected widen-before-resolve because it spends the "never briefly widen for an unvetted host" property. Coda: the reversible env-var door-opener evaporated across the launcher's UAC self-elevation hop — "a reversible knob is only reversible if it survives the path the user actually takes to turn it." `[failure-story][governance/security][engineering]`

4. **2026-06-10 — A third failure posture, and the index that lives only in RAM** — In-memory FTS5 lexical index (rebuilt at DEK-unlock) chosen over on-disk FTS5 precisely because an on-disk plaintext index is a shadow of the ciphertext it defeats. The same-day adversarial addendum found a CRITICAL: `submit_pending`'s dedup-DELETE lived in an open implicit sqlite transaction and the *next healthy commit* silently destroyed an unrelated pending document; fixed structurally (all checks + encryption before any DML, inside `with self._conn:`), plus audit-first inversion (an unauditable action never takes effect). `[engineering][failure-story][governance/security]`

5. **2026-06-14 — The images that may never load: building a feature welded shut** — Built a security feature whose entire current job is to refuse to act, proven by tests that assert *nothing happens* (an exploding fetch stub never called). The failure kept on record: a `![x](javascript: alert)` alt-text bypass sailed through three layers of green tests because the escaper's URL grammar `[^)\s]` disagreed with the WinUI renderer's `[^)]` by one whitespace class. Lesson that generalizes: *a sanitizer has two failure directions* (dangerous-survives AND benign-destroyed — the escaper was silently rewriting legitimate mailto:/#anchor links to about:blank), and a review walking only one is half a review. `[failure-story][governance/security][engineering]`

6. **2026-06-11 — Catching the subclass while the base class escaped** — One hostile 64KB frame with a ~65,200-char `request_id` killed the whole guest parser listener: `_send_error` caught `ParseChannelError` but `MessageFramer.encode` raised the *plain* `ValueError` base class — the catch was narrower than the raise surface. The stinger: a `# pragma: no cover — cannot overflow` comment asserting the path unreachable is exactly what kept coverage tooling from ever pricing it. "A pragma on an except clause is a claim, and claims about 'cannot happen' paths deserve adversarial pricing, not exemption from it." `[failure-story][engineering]`

7. **2026-06-11 — The capability my own runtime was pinned below** — The runtime venv is Python 3.11.9 (pinned for OpenVINO/14B validation); `socket.AF_HYPERV` first shipped in CPython 3.12. The runtime was pinned one minor version *below* a capability its own new feature needed. Rejected both the whole-stack 3.12 migration (re-validate everything to buy one socket constant) and hand-marshalling the SOCKADDR_HV via ctypes; shipped an out-of-process bridge running under 3.14 as a short-lived subprocess, isolating the version jump to one process boundary that self-deletes when the runtime catches up. `[engineering]`

8. **2026-06-12 — The pathway I was sure of, and the door that was already built** — The LA caught the session handing him a technical decision (which adjudication transport to use) and corrected it back: *"he's a novice; choosing between two adjudication transports is exactly the technical-implementation call he relies on me for, not a governance fork."* The session then owned the call with conviction — and was *confidently wrong* (no per-CAR vsock path exists; adjudication runs in-process). Only disk-verification-before-building caught it. Twin lesson: own the technical call AND verify it. `[human-AI-collaboration][failure-story]`

9. **2026-06-13 — The runbook that read clean and lied twice** — A runbook that passed review carried two latent host/guest-boundary defects that only the real Hyper-V console could reveal: rebuilding the ISO in place drops the per-VM read ACL Hyper-V silently stamps on attached media (`0x80070005`), and a cold busybox guest won't auto-probe the CD filesystem (`mount -t iso9660 -o ro /dev/sr0` required). "The console is the test those mocks can't be." `[failure-story][engineering]`

10. **2026-06-16 — Teaching the box to draw, with the door welded shut** — UC-010 image generation: the first capability that was *never one of the nine* original use cases (scope expansion, written down everywhere it touches so future-me doesn't mistrust it). The keeper is the content-safety posture: rejected bolting on a classifier as "security theater" for the legal boundary that actually matters, and wrote *"robust local technical control here is not achievable"* into the security ADR — "a comforting fiction in a security document is worse than a named, accepted, structurally-bounded risk." `[governance/security][engineering]`

11. **2026-06-10 — Two sessions wrote the same amendment, and the merge had to make them one law** — Two parallel sessions each authored "Amendment 1" to ADR-027 (one the *mechanics*, one the *policy*), blind to each other; git placed both full sections 50 lines apart with only the Status line flagged as conflict. Unified rather than renumbered (renumbering would falsify LA-ratified provenance). Then the governance lens *failed the session's own merge* — a "reconciled" label sat on a still-half-reconciled precondition. "A half-reconciled precondition wearing a 'reconciled' label in the SSOT is worse than an unreconciled one." `[process][governance/security][failure-story]`

12. **2026-06-13 — Sizing the parser VM, and the maximum that costs nothing** — A Dynamic-Memory measurement nearly lied: a 20-second idle window showed the exact signature of a missing balloon driver (assigned pinned at startup, demand flat zero) — but the driver was simply ~60s slow to engage, and the deliberately-designed 60s sustained-parse burst outlived the transient and rescued the conclusion (peak ~256MB, floor 512MB, ~1.5GB handed back). Community-grade performance capture; the measurement's job turned out to be confirming the config was already optimal. `[engineering][community/OSS]`

13. **2026-06-10 — The VM that owned itself out of being stopped** — An ownership ratchet: the launcher only stopped the VM `if _vm_was_started`, a sticky flag that latches *off* forever after any single transient leak (crash, hard close, host reboot), so every subsequent clean exit skips the stop and the 2GiB leak self-perpetuates. Fixed by deciding from the VM's live state, not from who started it; unrecognised policy resolves to always-stop, never to a leak. `[failure-story][engineering]`

14. **2026-06-11 — The deck that taught its own lesson 103, then needed three tries to measure "readable"** — The lessons-teaching deck shipped in the morning and was stale by evening (its 12 cited fragment files deleted at fold-in). The three-try arc to measure "readable" is the gem: v1 counted below-the-fold ink as spillage (false positives); v2 found a real escape (`flex-shrink:1` crushing a diagram card); v3 confronted that "fits" had scaled a 9-node chain to 0.34 — green and unreadable. "The check was measuring the mechanism when the property that matters is a human can read it." `[process][failure-story][human-AI-collaboration]`

15. **2026-06-14 — The toggle that promises RAM back, and the delete button that was already built** — Honesty about the limit of headless verification: shipped voice load/unload toggles but flagged *before building* that it cannot prove the C++/OpenVINO/ONNX driver layer actually returns device memory — verified only that no lingering Python reference makes the unload a structural no-op, and left the PERFORMANCE_LOG results table deliberately blank rather than infer numbers. `[engineering][process]`
