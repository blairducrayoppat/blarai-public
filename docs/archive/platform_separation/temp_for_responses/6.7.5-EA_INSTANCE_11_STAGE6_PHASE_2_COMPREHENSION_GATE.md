## CG6 — A33 standing workaround acknowledgment

`mcp__vikunja__complete_task` is **PERMANENTLY SUSPENDED** in Phase 2 and all subsequent phases until Ticket WW (Vikunja id 318) is resolved.

Root cause: A33 data-loss bug — `complete_task` sends only `{"done": True}`; the Vikunja Go backend zero-fills absent fields on partial update, blanking `description=""` and `priority=0` (and silently emptying `title` if not preserved).

**Option B workaround applied to every close** (Phase 2 tracking task and any incidental closes):

1. `mcp__vikunja__get_task(task_id)` — capture `title`, `description`, `priority`.
2. `mcp__vikunja__update_task(task_id, title=t, description=d, priority=p, done=True)` — pass all four explicitly. Both `done=True` and `title` MUST be explicit:
   - `done=True` explicit — Go zero-value `false` would reopen the ticket.
   - `title` explicit — required field; omitting silently empties the title.
3. Verify post-close: `mcp__vikunja__get_task(task_id)` → `done=true`, `description` preserved, `priority` preserved, `title` preserved.
