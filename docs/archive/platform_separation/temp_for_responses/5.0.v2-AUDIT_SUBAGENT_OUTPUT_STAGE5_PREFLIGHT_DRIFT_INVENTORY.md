**FINDING**: Confirmed REAL. If move 1 succeeds (sprints), move 2 succeeds (scheduled), move 3 fails (active_tasks.yaml destination unexpectedly exists due to race condition), the loop throws. State at throw time: 2 moves applied, 2 not. No rollback logic.

**RISK SCENARIO**: EA-9's PowerShell session fails mid-loop (e.g., disk full, permission error on one of the moves). Working tree is in an inconsistent state. Manual recovery = move-back the partial moves; non-trivial under stress.

**RECOMMENDED DISPOSITION**: BIND under `g9-ea9_n1(m)` — wrap the foreach in a try/catch with rollback:
```powershell
$applied = @()
try {
  foreach ($m in $moves) {
    if (-not (Test-Path $m.From)) { Write-Host "SKIP (missing): $($m.From)"; continue }
    if (Test-Path $m.To) { throw "Destination already exists: $($m.To)." }
    $parent = Split-Path $m.To -Parent
    if (-not (Test-Path $parent)) { New-Item -ItemType Directory -Path $parent -Force | Out-Null }
    Move-Item -Path $m.From -Destination $m.To
    $applied += $m
    Write-Host "MOVED: $($m.From) -> $($m.To)"
  }
} catch {
  Write-Warning "Move loop failed mid-flight: $_"
  Write-Warning "Rolling back $($applied.Count) applied moves..."
  foreach ($m in $applied) {
    if ((Test-Path $m.To) -and (-not (Test-Path $m.From))) {
      Move-Item -Path $m.To -Destination $m.From
      Write-Host "ROLLED BACK: $($m.To) -> $($m.From)"
    }
  }
  throw "Item 5.3 atomicity broken; rolled back applied moves. Re-investigate before retry."
}
```
