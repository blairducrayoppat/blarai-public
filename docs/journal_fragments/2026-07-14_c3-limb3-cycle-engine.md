### 2026-07-14 — A pure engine's honesty lives in its failure vocabulary, not its happy path

*Plain summary: The C3 heartbeat's cycle engine (limb 3 of 7) merged dormant — one wake cycle as a pure function over injected substrate, where the hard design work turned out to be naming what happens when each piece is absent, corrupt, or lying.*

The cycle engine (`shared/coordinator/heartbeat_cycle.py`, merged `644c6f40`, branch commits `66789b24` + `8b80c5d9`) is the largest C3 limb: the design §2 orchestration — mode ladder, TTL sweep, snapshot compose, board-history observe, run harvest, stall pass, redispatch staging, tripwire, drafting seam, digest — as one function over an injected environment. Fifty-eight lock tests, and almost none of them test the happy path. That inversion is the lesson. When I listed the acceptance shape before writing code, roughly four in five locks were about degraded states: the swap state that cannot be read (never clearance to draft), the board that reads UNREACHABLE (never observed, so a project's age history survives an outage untouched), the acceptance record that names a path outside the workspace (staging skipped, surfaced, never "recovered" from run text), the store that is not there at all (the deterministic organizing proceeds without it). Writing the failure vocabulary first — every `SurfacedCondition` kind, and the `machinery_health` bit that marks which conditions may never be shadow-gated — made the happy path fall out almost incidentally.

The independent review (author ≠ verifier, verdict MERGE-READY-WITH-NITS) earned its keep in exactly this territory, with both mandatory dimensions — the design-§5 trusted repo_id sourcing and the #844 c.1876 defensive wrap — verified clean, and three MINOR findings that were all subtle lies my code told about its own recovery behavior. The best one: a comment claiming a failed DRAFT→STAGED promotion would be "retried next cycle." It would not have been. The stager's dedup would find the stranded DRAFT, report it as already-known, and the promotion loop — which only iterated freshly-staged proposals — would never look at it again; the proposal was immortal and invisible. The fix rides on a discriminator that was sitting in the store all along: a never-surfaced DRAFT has an empty `staged_at`, a TTL-demoted one does not, so the engine can re-promote the first without re-nagging the second. The second finding of the same species: my absence-exit comment called a failed stamp-unlink "idempotent-cost" when the extension was actually additive — a stuck Windows file lock would have compounded the TTL pause every fifteen minutes until proposals never expired. An `applied_at` marker in the stamp now makes the extension exactly-once per absence episode. Trade-off named: I rebuilt a corrupt board-history record from the current cycle's observations (surfaced, ages reset to cold start) rather than preserving the corrupt bytes for forensics — a permanently degraded record that only operator vigilance would notice seemed the worse failure; the rejected path was refusing to write until a human intervened.

One reviewer finding was a scoping error in the conservative direction: my tripwire suppressed on ANY unreachable substrate, including the board-history age record — but the age record has nothing to do with counting Ready work, so a torn local JSON file would have silenced the exact "work is waiting and nothing is pulling" alarm the tripwire exists to raise, mislabeled as a Vikunja outage. Suppression is now scoped to the substrates the predicate actually consults. The general lesson, third time this program has paid for it: *conservative in the wrong dimension is not conservative* — an over-broad suppression is a fail-open on the alarm's duty of candor.

Limbs 4 (shadow journal + router) and 5 (drafting adapter) built in parallel worktrees while this landed — the review-overlapped-with-next-build cadence held for the first time across three units, and the wall-clock saving is real (limb 5's review surfaced its one LA decision point — a chat turn can wait behind an in-flight draft once live — before limb 6 even started, which is when it is cheap to carry).
