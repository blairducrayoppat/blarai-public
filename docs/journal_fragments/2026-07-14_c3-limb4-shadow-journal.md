### 2026-07-14 — A routing table is a security boundary when one row must never obey the mode flag

*Plain summary: The C3 heartbeat's shadow journal and output router (limb 4 of 7) merged dormant — every coordinator side effect now flows through one routing table whose most important row, machinery health, is structurally incapable of being shadow-silenced.*

Limb 4 (`shared/coordinator/shadow_journal.py` + `shared/coordinator/output_router.py`, built by a worktree agent at `e847e8ee`, merged to main behind an independent MERGE-READY-WITH-NITS review) realizes the design's §7.2 routing table: in shadow mode every stall comment, board move, proposal copy, digest, and tripwire alarm lands in a born-encrypted journal instead of in front of the operator — except machinery health, which reaches the operator surface in BOTH modes, always. The judgment worth recording is HOW that exception is enforced. A flag check (`if not shadow_mode or is_health:`) would have been one inverted condition away from routing the dead-man's own alarm into an unread journal — the exact vigilance-dependence failure the design's §2.14.1 exists to kill. Instead the health path is a separate method that never reads the mode flag at all, locked by a test that inspects the compiled function's variable names, and `machinery_health` is deliberately excluded from the journal's kind allowlist — the watchdog's alarm is not just not-routed to the journal, it is UNJOURNALABLE. Structural absence over configuration, applied to a code path rather than a feature: the strongest form of "this row ignores the flag" is a row that cannot see the flag.

Two smaller judgments. First, `shadow_mode` inverts the codebase's own fail-closed idiom: every other boolean in `CoordinatorConfig` resolves missing-or-mistyped to False (off is safe), but for this key False means LIVE OUTPUT — so it resolves missing-or-mistyped to True, and only a genuine TOML boolean `false` (post-graduation, LA-ceremonied) goes live. Same principle, opposite polarity; the rejected path was reusing the shared `_bool` helper because it was there. Second, the journal-fault asymmetry: when the journal write fails, effects with a natural retry (a stall comment whose seen-set entry is only persisted on success; a board move re-derived next cycle; a proposal already durable in the store) fail soft, while ephemeral effects with no other record (a digest, a tripwire alarm) fail LOUD through the health surface — a lost journal write may cost evidence, but it may never silently suppress the only copy of an alarm.

The review's three findings all landed as obligations on later work rather than code changes here, which is itself the parallel-build cadence paying off: the reviewer traced the router's contracts INTO the already-merged cycle engine and OUT to the unbuilt launcher wiring, and what it found were seams limb 6 must not drop (route every surfaced condition through the one safe entry; journal a copy of each shadow-staged proposal) plus a grading caveat for #855 (shadow board-move attribution keys on the run, never the project). Those went onto the ticket in the same motion as the merge — durable queue, not session memory.
