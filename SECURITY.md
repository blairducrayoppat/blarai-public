# Security Policy

This repository is a **snapshot publication** of a privately developed project.
Development happens in a private repository; public snapshots are published
periodically after automated secret/content gating.

## Reporting a vulnerability

Please report suspected vulnerabilities **privately**:

- Preferred: GitHub's **"Report a vulnerability"** button (Security tab of this
  repository — private vulnerability reporting is enabled), or
- Email: **mr.blair.do@gmail.com** with subject line `SECURITY`.

Please do not open public issues for security reports.

You should receive an acknowledgment within a few days. Because this is a
personal project maintained by one person, there is no bug-bounty program —
but reports are genuinely appreciated and will be credited in the fix's
release notes unless you prefer otherwise.

## Supported versions

Only the **latest public snapshot** (the tip of `main`) is in scope. Older
snapshot commits are historical records and do not receive fixes.
